# 2026-02-28 進捗ログ

## 取り組んだこと

- `HashMap` を保持する `Storage` 構造体を設計し、定義した。
- `Storage` 構造体に、パース済みの `Command` を実行する `execute` メソッドを実装した。
- `match` 式を使い、`Command::Set` と `Command::Get` の処理分岐の骨格を作成した。
- `Command::Set` のロジックを実装し、`HashMap` にデータを挿入できるようにした。

## 学んだこと・要点

1.  **構造体と実装ブロック (`impl`)**: データ（`struct`）と、そのデータを操作する振る舞い（`impl` 内のメソッド）をセットで定義する、Rustの基本的な設計パターンを学んだ。
2.  **可変参照 (`&mut self`)**: 構造体自身のデータを書き換えるメソッドを定義する際は、第一引数に `&mut self` を取る必要があることを確認した。これにより、メソッドは `Storage` の `data` フィールドを安全に変更できる。
3.  **データ付き `enum` のパターンマッチ**: `match cmd { Command::Set { key, value } => ... }` のように、`enum` のバリアントが持つデータをパターンマッチで直接取り出せることを実践した。
4.  **`HashMap` の基本操作**: `HashMap::insert()` を使ってキーと値を保存する方法を実装した。
5.  **戻り値の設計**: `GET` のように「値がある場合」「ない場合」「エラーの場合」を表現するために、`Result<Option<String>, String>` のような複合的な型が有用であることを確認した。
